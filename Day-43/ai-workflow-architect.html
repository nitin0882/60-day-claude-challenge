Workflow Summary

This workflow takes a Next.js e-commerce build from a blank repo to a launched, revenue-generating store. It front-loads planning and data modeling (stages 1–3) so the product catalog, cart, and checkout (4–5) can be built on stable foundations, treats Next.js API routes and server actions as the backend (6), and closes with the QA, deployment, and optimization work that determines whether the store actually converts (7–9). AI tools are used as accelerants at each step — for scaffolding, copywriting, code review, and test generation — not as a replacement for the architectural decisions a developer still needs to make.
